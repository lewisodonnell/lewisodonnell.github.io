# lewisodonnell.github.io

Personal academic website. [Astro](https://astro.build), fully static output,
vanilla CSS with design tokens, and a single small inline script (no
frameworks) for the theme toggle, scroll reveals, self-drawing wave dividers,
3D card tilt + spotlight, hero parallax, nav state, and scroll-spy. Light and
dark mode follow the system by default; the nav toggle overrides and is
remembered. Served from <https://lewisodonnell.github.io>.

## Run locally

```sh
npm install
npm run dev        # http://localhost:4321
npm run build      # static output in dist/
npm run preview    # serve dist/ locally
```

## How deployment works

Every push to `main` triggers `.github/workflows/deploy.yml`, which builds the
site and publishes `dist/` to GitHub Pages (Pages source: **GitHub Actions**).
There is nothing to do manually — commit, push, wait ~1 minute.

The production URL lives in exactly one place: `site` in `astro.config.mjs`.
The sitemap, RSS feed, canonical URLs, OG tags, robots.txt and JSON-LD all
derive from it.

## Adding content

**A publication** — add an entry to `src/data/publications.yaml` (title,
authors, venue, year, links). Optional thumbnail: drop an image into
`public/thumbnails/` and set `thumbnail: /thumbnails/name.png`. Done.

**A project** — add an entry to `src/data/projects.yaml`; optional image in
`public/projects/` (set `imageAlt` when you add one).

**A news item** — add an entry at the top of `src/data/news.yaml` with
`date: YYYY-MM` and `text` (inline HTML links allowed).

**A blog post** — create `src/content/blog/my-post.md` with frontmatter
`title`, `date`, `summary` (and optionally `draft: true` to keep it out of
the published site). Math via `$…$` / `$$…$$`, fenced code blocks are
highlighted. The post appears on `/blog`, in the homepage Blog section,
and in `/rss.xml` automatically.

## Assets dropped in by hand

- `public/headshot.jpg` — the About photo appears automatically once this exists.
- `public/cv.pdf` — the CV link (header + footer) appears automatically once this exists.
- `public/og.png` and `public/apple-touch-icon.png` are generated by
  `npm run og` (re-run only if the name or tagline changes, then commit).

Remaining content placeholders: `grep -rn "FILL" src/`

## Custom domain later

1. Buy the domain.
2. In this repo: **Settings → Pages → Custom domain**, enter the domain and
   save (GitHub commits a `CNAME` file).
3. At your registrar, set DNS per
   [GitHub's docs](https://docs.github.com/en/pages/configuring-a-custom-domain-for-your-github-pages-site/managing-a-custom-domain-for-your-github-pages-site):
   - `CNAME` record for `www` → `lewisodonnell.github.io`
   - `A` records for the apex → `185.199.108.153`, `185.199.109.153`,
     `185.199.110.153`, `185.199.111.153`
     (optionally `AAAA` → `2606:50c0:8000::153` … `:8003::153`)
4. Change the single `site` value in `astro.config.mjs` to the new domain and
   push.
5. Once the certificate is issued, tick **Enforce HTTPS** in Pages settings.

The old `lewisodonnell.github.io` URL redirects to the new domain
automatically, so existing links keep working.
